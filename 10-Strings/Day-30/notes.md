## Implement ATOI

### Key Idea
Handle edge cases sequentially: strip leading whitespaces, check for an optional `+` or `-` sign, and then iterate through characters as long as they are digits. Accumulate the result and finally clamp it to the 32-bit signed integer range `[-2^31, 2^31 - 1]`.

### Pattern
String Parsing

### Complexity
Time: O(N)
Space: O(1)

### Revision Note
Pay close attention to integer bounds and handle the sign properly.

## Implement strStr (Find the Index of the First Occurrence in a String)

### Key Idea
The simplest way is to slide a window of the `needle`'s length across the `haystack` and check for substring equality. (Note: Rabin-Karp or KMP can do this faster, but substring slicing in Python is highly optimized in C).

### Pattern
Sliding Window (String Match)

### Complexity
Time: O(N * M) worst case, O(N) on average with Python's optimized slicing
Space: O(1)

### Revision Note
Checking `if haystack[i:i+m] == needle` is often the most practical solution in Python unless you are specifically asked to implement a complex string matching algorithm like KMP.

## Longest Common Prefix

### Key Idea
Find the shortest string in the array, as the common prefix cannot be longer than it. Iterate through each character of the shortest string, comparing it against the character at the same index in all other strings. The moment there's a mismatch, return the prefix up to that point.

### Pattern
Vertical Scanning (String iteration)

### Complexity
Time: O(S) where S is the sum of all characters in all strings
Space: O(1)

### Revision Note
Finding the shortest string first is an optimization that prevents out-of-bounds errors and limits the outer loop.

## Rabin-Karp Algorithm

### Key Idea
A string-matching algorithm that uses hashing to find any one of a set of pattern strings in a text. Calculate the hash of the pattern and the first window of text. Then, slide the window by removing the leading character's contribution to the hash and adding the new trailing character's contribution (rolling hash). Only check for exact character matches if the hashes are equal.

### Pattern
Rolling Hash (String Match)

### Complexity
Time: O(N + M) average case, O(N * M) worst case (if hashes collide frequently)
Space: O(1)

### Revision Note
The magic formula for the rolling hash shift is `t = (d * (t - ord(text[i]) * h) + ord(text[i + m])) % q`. Adding `q` if `t < 0` handles negative values generated by modulo operations in some languages.

## Day Takeaway
String manipulation heavily involves checking boundaries, dealing with encodings (like ASCII characters via `ord`), and matching substrings either naively or using optimized hash-based algorithms like Rabin-Karp.
